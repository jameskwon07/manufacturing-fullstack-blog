# 주니어 개발자가 듣고 싶은 제조 풀스택 주제

블로그의 기본 관점:

> "내가 왜 이 길을 선택했는가"보다 "주니어 개발자가 이 분야를 이해하려면 무엇을 알아야 하는가"를 먼저 쓴다.

## 제목 원칙

- "나"보다 "주니어 개발자", "제조 앱", "풀스택", "설계", "데이터 모델링"을 앞에 둔다.
- 회고형보다 문제 해결형 제목을 우선한다.
- 검색할 만한 단어를 제목에 넣는다.
- 한 글은 하나의 질문에 답한다.

## 좋은 제목 예시

- 주니어 개발자가 제조 도메인 풀스택을 공부하면 좋은 이유
- 제조 앱은 일반 웹 서비스와 무엇이 다를까
- MES를 처음 공부하는 개발자를 위한 아주 얕은 소개
- 작업지시, 공정, LOT를 데이터베이스로 어떻게 표현할까
- 제조 시스템에서 이력 관리가 중요한 이유
- 엑셀로 하던 제조 업무를 웹 앱으로 바꿀 때 먼저 봐야 할 것
- 제조 현장용 관리자 페이지를 설계할 때 조심할 점
- 품질 검사 데이터를 저장할 때 필요한 테이블들
- 설비 점검 앱을 만든다면 어떤 기능부터 설계해야 할까
- 주니어 개발자를 위한 미니 MES 포트폴리오 설계
- 제조 앱에서 권한 설계가 어려운 이유
- 제조 데이터 모델링을 공부할 때 알아야 할 기본 용어
- 현장 사용자를 위한 입력 화면은 어떻게 달라야 할까
- 제조 대시보드는 어떤 지표를 보여줘야 할까
- 풀스택 개발자가 업무 프로세스를 먼저 그려야 하는 이유

## 독자가 궁금해할 질문

- 제조 회사에 가면 소프트웨어 개발자는 무슨 일을 하나?
- 하드웨어를 잘 몰라도 제조 도메인 개발을 할 수 있나?
- MES, ERP, QMS, WMS, SCADA는 어떻게 다른가?
- 제조 도메인 개발자는 어떤 데이터베이스 설계를 많이 하나?
- 포트폴리오로 제조 앱을 만들면 어떤 주제가 좋나?
- 일반 SaaS 개발과 제조 내부툴 개발은 무엇이 다른가?
- 주니어가 제조 도메인에 들어가기 전에 무엇을 공부하면 좋나?
- 업무형 UI는 토이 프로젝트 UI와 무엇이 다른가?
- 제조 시스템에서 로그와 이력은 왜 중요한가?
- 설계 문서 중심 포트폴리오는 어떻게 만들면 좋나?

## 피해야 할 제목

- 나는 왜 제조 분야 개발자가 되고 싶은가
- 내가 생각하는 좋은 개발자
- 나의 커리어 방향
- 내가 공부한 것들
- 제조 회사에서 느낀 점

이런 제목은 작성자의 사정이 앞에 나온다. 독자 입장에서는 "그래서 나에게 어떤 도움이 되지?"가 늦게 온다.
